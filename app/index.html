Creating a Heroku Blog App

Written by MJ Grant on August 11, 2014.

This blog is hosted on Heroku. Here's how it was created:


  1. Create a new repo on Github
  2. Clone the repo onto your local environment
  3. Do your initial commit, then create a branch: git checkout -b workingbranch
  4. Follow this tutorial to make the actual blog:
  5. Make a heroku.com account
  6. Log into Heroku.com and set your Dynos to at least 1
  7. Run grunt build
  8. Follow these steps to push to Heroku: https://devcenter.heroku.com/articles/git#deploying-code
  9. Commit and push the blog to a branch on Github
  10. Make sure /dist/ is not in .gitignore
  11. Edit server.js: '/app/' should be '/dist'
  12. Push to Github
  13. Push to Heroku: git push heroku workingbranch:master
When grunt serve is running, the blog will be automatically updated in the browser as you work on it. This "quick preview" feature is particularly helpful when building the stylesheet.

The Case of the Missing CSS

Written by MJ Grant on August 12, 2012.

This site was missing its css styles for a while. Luckily, the fix was simple:

  1. run grunt build
  2. In server.js, change '/app/' to '/dist'.
